# ビクーニャ（Vicuna Reserve）

LAMAライクなカードゲームに予約システムを追加した新感覚ゲームのデジタルプロトタイプ。
ボドゲ会の中量級セッションでのバランス検証がメイン用途。
Vite + React + TypeScript + Tailwind CSSで実装。

## セットアップ・起動

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # 型チェック（tsc -b） + 本番ビルド（vite build）
npm run preview  # ビルド後のプレビュー
npm run lint     # ESLint（型チェックは`npm run build`のtsc -bで実施）
```

## 構成

- `src/engine/`: 純ロジック（React非依存）
  - `types.ts`: Token/Player/GameState等の型定義
  - `deck.ts`: デッキ生成・シャッフル・出せる札判定
  - `rules.ts`: ラウンド状態遷移（play/draw/fold/reserve/pass、勝敗判定）
  - `ai.ts`: AI意思決定ロジック（`smart`＝手札の質で選別 / `naive`＝固定確率ベースライン）
  - `simulation.ts`: CvC統計シミュレーション（1試合実行・バッチ実行）
- `src/components/`: 共通UIパーツ（`Card`, `ChipRow`, `PlayerBadge`）
- `src/screens/`: 画面ごとのコンポーネント
  - `MenuScreen`: モード選択
  - `HotseatGame`: 対人戦（1台の端末を回して遊ぶ）
  - `CvcWatch`: CvC観戦（1手ずつ進行 or 自動再生）
  - `CvcStats`: CvC統計シミュレーション（ペナルティ値・AIロジック比較）
- `src/App.tsx`: モード切り替えのオーケストレーション層

## ゲームルール概要（解釈B、確定済み）

- 予約は各プレイヤーの最初の手番のみ宣言可。1ラウンドにつき1人まで（早い者勝ち）
- 予約中のプレイヤーはそのラウンド中、降りられない
- 予約者が手札を出し切って上がる（成功）→ チップ2枚を返却（通常は1枚）
- 予約者が上がれなかった（失敗）→ 白チップ+2/+5/+10のペナルティ（値は調整中）
- 投了は本来のLAMAにないルール。予約者が「山札切れ＋出せるカードなし」で手詰まりになった時だけの救済アクション（実質は予約失敗と同義）。他に1人以上アクティブなプレイヤーがいる場合のみ発生しうる（自分以外全員が降りていれば、その時点でラウンドは自動終了するため）

## 運用ルール

### ファイル名にバージョン番号をつけない

版数はファイル名ではなくgitのコミット履歴・ブランチで管理する。

### バランス変更はシミュレーションで検証してから確定する

1. `src/screens/CvcStats.tsx`（CvC統計シミュレーションUI）で効果を確認してから、ペナルティ値やAIロジックの変更を確定する
2. AIロジックの効果測定は`smart`（現行）と`naive`（固定確率ベースライン）を比較し、
   「ルール自体が強いのか、AIの選抜が効いているのか」を切り分ける
3. 経済レバー（ペナルティ値等）は単一変数で動かす（同時に複数いじらない）
4. 平均値だけでなく分布（予約宣言率・予約成功率・予約成功者の最終勝率など）も見る
